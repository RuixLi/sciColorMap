# Artifact manifest format

The machine format for an artifact's provenance record: an auto-emitted JSON sidecar, and a generated index over the sidecars. A reference the `working-with-data` convention and future tooling lean on; non-normative here. `working-with-data` owns *when* a record is required (load-bearing artifacts); this describes its *shape*.

## The sidecar

`<artifact>.json` beside the artifact, carrying `"kind": "artifact-manifest"` and `"schema_version": 1` so it is identifiable by content, not filename. Auto-emitted by the producer (script, tool, or hook) at write time - a manifest hand-written after the fact rots.

Fields, by tier:

- **required identity** *(auto)* - `kind: "artifact-manifest"`, `schema_version`, `artifact` (path), `hash`, `created` (ISO date).
- **recommended provenance** *(producer-owned)* - `producer` (the script or program that created the artifact, not the manifest writer; script + hash and optionally version/commit), `inputs` (each: path + hash), `params` (path + hash for the run's `ops_*.json` or equivalent params file).
- **trust** *(optional)* - `ground` (authority | property | reference | inspection), `evidence`, `validated_by`, `validated_at`. Decay = today's `inputs` hashes differ from the recorded ones.
- **sanity** *(optional)* - expected `shape`, `range`, `dtype`, or other checkable properties.
- **semantic** *(optional)* - what it is, and a `legend` mapping each field / label / metric to its computation and units.
- **categorical** *(optional)* - `tags` for sorting and locating.
- **reserved optional metadata** - `environment` (path + hash records for pinned environment files) and `command` / `run` metadata may be added when recurring use proves the need.

Richness tracks the emitter: a hook supplies only `identity`; only the producing script can supply `provenance` (it alone sees what was read); the agent adds `semantic` / `sanity` / `tags` for meaning-search. Enrich - semantic and tags - only what you will search by meaning (the deliverables); intermediates stay identity+provenance and are reached through their producer.

```json
{
  "kind": "artifact-manifest",
  "schema_version": 1,
  "artifact": "data/interim/AM_cnt.mat",
  "hash": "sha256:...",
  "created": "2026-06-17",
  "producer": { "script": "scripts/process/make_am.m", "hash": "sha256:...", "version": "git:abc123" },
  "inputs": [{ "path": "data/raw/...", "hash": "sha256:..." }],
  "params": { "path": "configs/ops_make_am_20260617T1012.json", "hash": "sha256:..." },
  "trust": { "ground": "reference", "evidence": "test_make_am", "validated_by": "agent", "validated_at": "2026-06-17" },
  "sanity": { "shape": [52, 52], "range": [-1, 1], "symmetric": true },
  "semantic": { "is": "group-mean adjacency matrix (Fisher-z FC)", "legend": { "rows_cols": "Allen regions", "value": "Fisher-z correlation" } },
  "tags": ["genotype:cnt", "stage:interim", "kind:AM"]
}
```

## The index

A single index over all sidecars, regenerated by scanning (never hand-kept), so one file read locates any artifact at scale. Default home: `context/artifacts.json`. A digest, not a copy: keep the keys you filter, sort, or sweep on plus the pointers (`manifest`, `artifact`, `hash`, `inputs` path@hash, `producer`, `created`, `tags`, `is` - the one-line semantic); leave the full `legend`, `sanity`, `trust`, and params detail in the sidecar.

```json
[
  { "manifest": "data/interim/AM_cnt.mat.json", "artifact": "data/interim/AM_cnt.mat", "is": "group-mean adjacency matrix (Fisher-z FC)",
    "tags": ["genotype:cnt", "kind:AM"], "producer": "scripts/process/make_am.m",
    "created": "2026-06-17", "hash": "sha256:9f2a", "inputs": ["data/raw/ica_cnt.mat@sha256:11ab"] }
]
```

## Tooling (build from friction)

Implemented in the tools module:

- **index generator** - `index-artifacts` scans sidecars -> the index.
- **artifact checker** - `check-artifacts` fails only when the artifact itself is missing or its hash changed. Missing or changed recorded provenance (`inputs`, `producer`, `params`) and stale/missing indexes are warnings, because the artifact can still be present and unchanged. `sanity` blocks are not checked here; the framework cannot assume a data format.

The format stays tool-exportable so its provenance can later generate a `Snakefile` / `dvc.yaml`.
